Data Cleaning in SQL

In this project, raw sales data is imported into SQL Server, where it is cleaned, normalized, and transformed to improve structure and quality for in-depth analysis and reporting.

  • Input used : The input data is collected from CSV files, containing detailed information about raw sales transactions from Atlic Hardware. These files include key sales metrics such as product IDs, quantities sold, transaction dates, and customer information.
  • Tables :
    • sales transactions
    • sales customers
    • sales date
    • sales product
  • Tools Used : MySQL, Power BI, MS Excel
  • EDA :
    • The collected data is ingested into a SQL Server database, where it is stored in structured tables for analysis.
    • Exploratory Data Analysis (EDA) techniques are applied using SQL queries to explore and summarize the dataset.
    • These queries help in filtering, grouping, and aggregating the data to uncover underlying patterns and trends.
    • The results from the EDA are used to identify key features and insights for further analysis or decision-making.
  • Data Visualization :
    • After the data is processed and analyzed, it is imported into Power BI for visualization and reporting.
    • Various types of visualizations, such as bar charts, line graphs are used to represent key insights from the data.
    • Interactive tables and dashboards are created in Power BI, allowing users to explore the data dynamically.
    • These visualizations help in presenting complex data trends and patterns in a more understandable and actionable format for stakeholders.
